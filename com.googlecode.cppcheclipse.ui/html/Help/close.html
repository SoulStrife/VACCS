close()

The C library function close() closes a file descriptor so that it will no longer point to a file and can then be reused The prototype is

int close(int fd)

If fd references a file descriptor that is currently open (recently created by a call to open()), the resources associated with fd will be freed

One potential problem with close() is when programmers do not check the return value. It is possible that errors from the last call to write() are only reported when close() is called. If the programmer ignores this, this can potentially lead to loss of data.
