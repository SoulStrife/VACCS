fclose()

The C library function fclose() closes the underlying file descriptor so that it will no longer point to a file and can then be reused. All buffers are also flushed. The prototype is

int fclose(FILE *stream)

If stream references a file stream that is currently open (recently created by a call to fopen(), fdopen(), or freopen()), the resources associated with stream will be freed

One potential problem with fclose() is when programmers do not check the return value. It is possible that errors from the last call to fwrite() are only reported when fclose() is called. If the programmer ignores this, this can potentially lead to loss of data.
