memmove()

The C library function memmove() copies a region of memory from one location to another. However, unlike memcpy()regions may overlap. The prototype is

char *memmove((char * dst, const char * src, size_t len)

Copying takes place as thought the bytes from src are first copyed to temporary space that does not overlap either src or dest. Then, the bytes are copied from the temporary space to dest. The variable len allows the programmer to specify how many bytes to copy. Unfortunately, there is no way to specify the length of the area allocated for dest. If dest is less than len bytes long, then memmove() copies extra bytes beyond the area allocated for src. In addition, if the memory for dest is smaller than len bytes, memory outside of the bounds of dest will be overwritten. No matter the size of memory for the source and destination len bytes are copied. If we consider the code below,

static char d[4];
static char s[6]]

int main(int argc, char **argv) {
  strncpy(s,"six12",6)
  memmove(d,s,7);
}
