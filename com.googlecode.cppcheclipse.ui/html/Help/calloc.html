calloc()

The C library function calloc() allocates a region of memory on the heap and initialzes that contents of that region to 0. The prototype is

void *calloc(size_t nmemb, size_t size)

If there is enough space available, calloc() will allocate size bytes of space.

One potential problems with calloc() is undefined behavior when size is 0. calloc() does not check if size is 0 and its behavior for allocating a 0-byte region is implementation dependent. This means that it will depend upon the system and compiler as to what the behavior of calloc is when a 0-byte region is requested.

In addtion, if more space is requested then is available, a call to calloc() may succeed. However, the process will be killed if an attempt is made to access that region. For example, 

int main(int argc, char **argv) {
    char *p = calloc(INT_MAX);
    *p = 'a';
}

may succeed on the call to calloc() but be killed when *p is assigned.
