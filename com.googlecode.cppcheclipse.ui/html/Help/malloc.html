malloc()

The C library function malloc() allocates a region of memory on the heap. The prototype is

void *malloc(size_t size)

If there is enough space available, malloc() will allocate size bytes of space.

One potential problems with malloc() is undefined behavior when size is 0. malloc() does not check if size is 0 and its behavior for allocating a 0-byte region is implementation dependent. This means that it will depend upon the system and compiler as to what the behavior of malloc is when a 0-byte region is requested.

In addtion, if more space is requested then is available, a call to malloc() may succeed. However, the process will be killed if an attempt is made to access that region. For example, 

int main(int argc, char **argv) {
    char *p = malloc(INT_MAX);
    *p = 'a';
}

may succeed on the call to malloc() but be killed when *p is assigned.
